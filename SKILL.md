# pure-skill

## When to use

当用户要求净化项目，提纯项目，pure 时调用

## Instructions

1. 询问用户该项目的主要业务逻辑，用户可以直接跳过，跳过之后由项目获取 `README.md` 文件或者 `CLAUDE.md` `AGENTS.md` 了解项目的大致逻辑；
2. 根据项目代码分析主要业务逻辑，并绘制数据流图标识出项目的操作逻辑
3. 根据输出产物所描述的，拆分多个 subagent 去统一生成整体的业务逻辑架构
4. 在生成业务逻辑期间，用户大概率也是不了解具体业务逻辑的，所以你必须启动多个 subagent 获取到更完善的架构逻辑

## 输出产物

所有输出产物统一放置在当前文件夹的 `PURE/`, 所有文件名及目录名使用大写

1. HANDOFF.md: 该文件包含整体项目的主要业务逻辑，并且支持在其它项目中直接复用
2. ARCHITECT/: 该目录下留存主要业务逻辑，并且可以使用 PlantUML 绘制整体数据流图，可以按照业务划分，并且罗列到 ARCHITECT/README.md
3. RULES/: 该目录下集成整体项目的开发逻辑规则及整体业务遵循的业务限定逻辑
4. OTHERS/: 该目录下留存各种项目支撑文件或代码逻辑

## Rules

1. 忽略项目 ignore 的项目以及所有隐藏文件，构建目录，开发环境目录
2. 所有业务逻辑必须进行脱敏操作，不得将真实的数据和表结构提炼出来，不得直接复用项目名称
3. 禁止直接复用原有的字典数据和字段名称，必须进行脱敏或者使用学名，专有名词除外
4. 不得体现原有数据和接口名称，不得体现原项目定义的路径名称或文件名称，必须使用专有名词替代
5. HANDOFF.md 要求技术栈解耦，不得直接复用当前项目技术栈，确保技术无关性
6. 不得有任何主观性的臆想行为，所有架构都必须有迹可循
